5) Security, compliance, and SEO governance

Baseline policies align with CSP/CORS enforcement, Search Console readiness, and cross-framework governance (NIST CSF, OWASP ASVS, CISA Cyber Essentials, PCI DSS 4.0).

Security headers + transport

  • CSP (default-src 'self', strict script/style policies)
  • HSTS (1-year, includeSubDomains, preload)
  • X-Frame-Options: DENY + frame-ancestors none
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy: disable sensitive APIs

CORS + data boundary controls

  • Access-Control-Allow-Origin: explicit allowlist
  • Access-Control-Allow-Methods: GET, POST
  • Access-Control-Allow-Headers: Content-Type, Authorization
  • Cross-Origin-Opener/Embedder/Resource-Policy set to same-origin
  • Minimize third-party scripts to reduce data exfiltration risk

SEO + Search Console readiness

  • Canonical URLs, OpenGraph/Twitter meta tags
  • XML sitemap + robots.txt with clear crawl rules
  • Google Search Console verification tag
  • Core Web Vitals + accessibility targets (WCAG 2.1 AA)
  • Structured data planned via JSON-LD (if CSP permits)

NIST / OWASP / CISA / PCI DSS alignment

  • Identify: asset inventory, vendor risk, data flow mapping
  • Protect: MFA, least privilege, encryption, patch cadence
  • Detect: logging + alerting (PCI DSS Req. 10)
  • Respond: incident runbooks + comms matrix
  • Recover: backups, continuity drills, MTTR tracking
OPS CyberSec Core: PCI DSS Requirements 3–11 mapped into Protect/Detect phases, reinforced with CISA Cyber Essentials and OWASP top risk controls.
