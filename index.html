2) Global layout + behaviors

2.1 Header (premium + minimal)

  • Left: logo
  • Center nav: Home, Operations, Call Center, IT Support, Professionals, Us
  • Right (optional): theme toggle + search icon
  • No Contact/Join/Chat in header (FAB-only)

2.2 Footer (simple, high-confidence)

  • Services: Operations / Call Center / IT Support / Professionals
  • Company: Us / Contact Us / Join Us
  • Help: Chat with Chattia (opens pop-up)
  • Bottom: “Remote support” + “We focus on organization, performance, and consistency.”

2.3 FABs (exact placement + behavior)

  • Position: bottom: 25px; right: 15px;
  • Stack: vertical, 12–14px gap
  • Contact → /contact-us
  • Join → /join-us
  • Chat → pop-up to Chattia
  • Small screens: collapse into “+” toggle
  • Hover/focus labels; keyboard + screen-reader ready

2.4 Chat (pop-up window)

  • window.open(CHATTIA_URL, "ChattiaChat", "width=420,height=720,noopener,noreferrer")
  • If blocked: toast with fallback “Open in new tab”
  • Brand rule: “Chat with Chattia (routing + questions)”

2.5 Premium rules

  • Max 6 sections per page
  • One primary CTA + one secondary link
  • Motion only for meaning
  • Single premium signature element (3D or glow grid)

4) Page-by-page wire outline

4.1 Home /

  1. Hero: “OPS Online Support — structured remote support for teams that need consistency.”
    • Subline: Operations, customer handling, IT continuity, professional support.
    • CTA: Explore Services → pillars
    • Secondary: How we work → /us
  2. Pillars (4 cards)
    • Operations: SOPs & routines, Coordination, Workflow clarity → /operations
    • Call Center: Intake & triage, Follow-ups, Scripts → /call-center
    • IT Support: Setup guidance, Troubleshooting, Secure practices → /it-support
    • Professionals: Admin support, Coordination, Documentation → /professionals
  3. How it works: Discover → Standardize → Support + CTA to /us
  4. What we improve: organization, consistency, documentation, follow-through + CTA to /contact-us
  5. Governance & Security: minimal data, clear boundaries, documentation-first + CTA to /us
  6. Final CTA band: “Tell us what you need supported…” → /contact-us

4.2 Operations /operations

  1. PageHero: “Operations support that stays organized under pressure.” + CTA /contact-us
  2. Capabilities grid: coordination, SOPs, scheduling, vendor/admin, summaries, quality checks
  3. Common scenarios: fall-through, no routines, needs clarity, docs used
  4. Deliverables: SOP pack, routine template, operating notes, escalation rules
  5. Working rhythm: cadence, channels, boundaries
  6. CTA band: “Describe your workflow…” → /contact-us

4.3 Call Center /call-center

  1. PageHero: “Customer handling built on clarity and consistency.” + CTA /contact-us
  2. Capabilities: intake, follow-ups, escalation, scripts, quality, multi-channel
  3. Standards system: script discipline, escalation boundaries, documentation-first
  4. Typical outcomes: cleaner handoffs, fewer missed follow-ups, consistent responses
  5. What you receive: scripts, follow-up rules, weekly notes
  6. CTA band: “Want consistent customer handling…” → /contact-us

4.4 IT Support /it-support

  1. PageHero: “IT support that keeps work moving.” + CTA /contact-us
  2. Capabilities: setup, access hygiene, troubleshooting, device support, backups, docs
  3. Access handling: minimum access, documented steps, avoid sensitive collection
  4. Deliverables: setup checklists, playbooks, standard docs
  5. Best-fit: growing team, recurring issues, remote continuity
  6. CTA band: “Tell us your stack…” → /contact-us

4.5 Professionals /professionals

  1. PageHero: “Professional support aligned to your workflow.” CTA /contact-us, link /join-us
  2. Roles: admin, operations coordination, customer support, documentation, project coordination
  3. Matching: scope → fit → start, communication standards, clear boundaries
  4. Standards: professional tone, documentation, security awareness, consistency
  5. CTA band: Contact Us + Join Us

4.6 Us /us

  1. PageHero: “Built on structure, clarity, and disciplined execution.” + CTA /contact-us
  2. Operating principles: documentation-first, routines, boundaries, minimal data, security habits
  3. How we work: Discover → Standardize → Support
  4. Governance: reduce access, disciplined handling, documented processes
  5. Who it’s for: SMBs, founders, teams
  6. CTA band: “Tell us what you’re running…” → /contact-us

4.7 Contact Us /contact-us

  1. PageHero: “Contact OPS Online Support” + CTA scroll to form
  2. Form: name, email, company optional, message, preferred method
  3. What to include: area, team size, improvement goals
  4. Chat option: Open Chat pop-up

4.8 Join Us /join-us

  1. PageHero: “Join OPS Online Support” + CTA scroll to form
  2. Who we’re looking for: communicators, documentation, follow-through, tone, security habits
  3. Application form: role, experience, languages, availability, timezone, portfolio
  4. What happens next: review → fit call → onboarding

5) Section system render contract

Switch on kind and render exactly as specified below.

5.1 Page render contract (global)

Page shape
{
  seo: { title, description },
  hero: { headline, subline, primaryCta, secondaryLink?, visual? },
  sections: Array<SectionSpec>
}

Render order
1) applySEO(page.seo)
2) <PageHero hero={page.hero} />
3) page.sections.map(renderSection)

5.2 CTA contract (shared)

Route CTA
{ label: "Contact Us", href: "/contact-us" }
Behavior: navigate to href

Scroll CTA
{ label: "Explore Services", action: "scrollTo", targetId: "pillars" }
Behavior: smooth scroll to id=targetId

Action CTA
{ label: "Open Chat", action: "openChatPopup" }
Behavior: open CHAT.url pop-up with CHAT.popup.features (fallback if blocked)

Rule: CTAs must never claim outcomes.

5.3 pillars

Props
{ kind: "pillars", id, title, items: [PILLARS...] }

Item shape
{ id, title, oneLine, bullets: [string], cta: { label, href } }

UI output
- Section title
- 4-card grid (responsive)
- Card: title, one-line, 3 bullets, CTA (route)

Premium
- Equal heights
- Hover-lift + subtle glow border
- Entire card clickable

5.4 steps

Props
{ kind: "steps", title, steps: [{ title, body }], cta? }

UI output
- Horizontal strip (desktop), stacked (mobile)
- Step numbers (01, 02, 03) or minimal icons
- Optional CTA below

Premium
- Subtle connecting line (desktop)
- No heavy illustrations

5.5 bullets

Props
{ kind: "bullets", title, bullets: [string], cta? }

UI output
- Section title
- Bullet list (max 6)
- Optional single CTA

Premium
- Short bullets (1 line)
- Minimal icons optional

5.6 trust (quiet trust)

Props
{ kind: "trust", title, bullets: [string], cta? }

UI output
- Same as bullets
- Softer glass panel
- Shield/check micro-icons optional
- Smaller headline style

Rule: no compliance claims, no "certified", no numbers.

5.7 capabilityGrid

Props
{ kind: "capabilityGrid", title, items: [string] }

UI output
- 2–3 column grid (responsive)
- Each item as capability pill card (icon optional, short text)

Premium
- 3–6 words per item
- Avoid paragraphs

5.8 cards

Props
{ kind: "cards", title, items: [{ title, body }] }

UI output
- Card grid of short scenario cards
- Title + 1–2 line body

Premium
- Keep bodies short
- Consistent heights
- No CTA buttons here

5.9 deliverables

Props
{ kind: "deliverables", title, items: [string] }

UI output
- List with check icons or compact tiles
- No downloads

Premium
- Grounded, tangible artifacts
- No promises

5.10 splitLane

Props
{ kind: "splitLane", title, left, right }

UI output
- Two side-by-side panels (stack on mobile)
- Left: Need support → Contact
- Right: Want to join → Join

Premium
- Same height panels
- One primary emphasis (left slightly brighter)

5.11 form

Props
{ kind: "form", id, title, fields: [{ name, label, type, required, ... }] }

Field types
text, email, url, textarea (rows), select (options[])

UI output
- Section title
- Glass panel form card
- Labels + required indicators
- Submit button

Premium
- Inline validation messages (gentle)
- Loading state on submit
- Success state replaces button row

Security
- Add spam protection later (Turnstile etc.)

5.12 ctaBand

Props
{ kind: "ctaBand", headline, primaryCta, secondaryCta? }

UI output
- Full-width band (contained max width)
- Headline + primary button
- Optional secondary link/button

Premium
- Conversion closer
- 1 sentence + 1 button

5.13 PageHero contract

Hero shape
{ headline, subline, primaryCta, secondaryLink?, visual? }

UI output
- Large headline
- Subline (max ~2 lines)
- CTA row (primary + optional secondary link)
- Optional visual (desktop right / mobile below)

3D rule
- Render 3D only if visual.enabled === true and device is suitable
- Otherwise render fallback

5.14 Global renderer behaviors

Section spacing
- Top/bottom: 48–64 desktop, 32–40 mobile
- Consistent max width (1100–1200px)

Card rule
- Shared radius, border, hover, typography

Motion rule
- On-scroll reveal for sections only
- Hover-lift on clickable cards + CTAs
- Respect reduced motion
